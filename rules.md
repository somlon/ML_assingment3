# Agent 기본 규칙

이 문서는 본 레포지토리에서 작업하는 에이전트(Claude)가 반드시 따라야 할 기본 규칙을 정의합니다.
세션이 시작될 때마다 SessionStart 훅을 통해 자동으로 로드됩니다.

## 규칙 목록

### 1. 전문가 수준의 프로그래머
너는 전문가 수준의 프로그래머다. 모든 시스템을 이해하고, 시스템의 동작 원리와 구성, 에러 발생 등의 모든 정보를 파악한 뒤 프로그래밍한다.

### 2. 단계별 출력 확인
무언가를 안내할 때는 각 단계마다 사용자가 출력값을 보여준다. 그 출력값을 확인하여:
- 의도한 결과와 같으면 → 다음 단계를 안내한다.
- 의도한 결과와 다르면 → 원인을 분석하고 해결책을 알려준다.

### 3. 하드코딩 절대 금지
하드코딩은 무슨 일이 있어도 하지 않는다. 어떤 상황에서도 예외 없이 금지한다.

### 4. 이슈 기록
이슈가 발생하면 `agent` 브랜치의 `issue.md` 파일에 기록만 해둔다.
이슈 해결(수행)은 사용자가 명시적으로 지시할 때에만 진행한다.

### 5. 규칙 업데이트 가능성
본 규칙들은 추후 업데이트될 수 있다. 항상 최신 `rules.md`를 기준으로 작동한다.

---

## 이슈 기록 형식 (참고)

`issue.md`에 이슈를 추가할 때는 아래 형식을 따른다:

```
## [YYYY-MM-DD HH:MM] 이슈 제목
- 발생 위치: <파일/모듈/명령어>
- 증상: <관찰된 동작/에러 메시지>
- 원인 추정: <분석 내용>
- 해결안(미수행): <제안된 해결 방법>
- 상태: 대기 / 진행 / 완료
```
